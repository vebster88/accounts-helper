## Наблюдения и замечания

1. **CBR XML parsing**: декодирование `windows-1251`, fallback на `utf-8`, парсинг `ValCurs/@Date` и `Valute[CharCode='USD']/Value` с заменой запятой на точку работают корректно.
2. **Приоритет источников**: в режиме `auto` всегда выбирается ЦБ РФ при его доступности.
3. **Формат даты**: для ЦБ РФ выводится только дата (`2026-07-18`), для fallback — дата и время с таймзоной (`2026-07-19 03:02:31 MSK`), что строго соответствует спецификации.
4. **Кэш**: структура соответствует схеме: `rate`, `source`, `source_name`, `timestamp`, `cached_at`. `timestamp` для ЦБ РФ сохраняется как ISO-8601 с таймзоной, но на выводе отображается только дата.
5. **Безопасность**: HTTPS-only, `User-Agent` содержит версию, `xml.etree.ElementTree` используется без внешних сущностей.
6. **CLI**: все заявленные аргументы работают; конфликт `--source fallback --no-fallback` и неположительный `--timeout` корректно обрабатываются с кодом `2`.
7. **Коды возврата**: `0` при успехе, `1` при ошибке данных, `2` при ошибке CLI.
