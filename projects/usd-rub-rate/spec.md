# Техническое задание (ТЗ): Скрипт отображения курса USD/RUB

## 1. Общие сведения

### 1.1. Цель документа

Настоящее техническое задание определяет требования к разработке консольного Python-скрипта `usd_rub_rate.py`, предназначенного для получения и отображения актуального курса обмена USD/RUB из публичных источников.

### 1.2. Основание для разработки

- Бизнес-требования: `brd.md`
- Высокоуровневое проектирование: `hld.md`

### 1.3. Язык и платформа

- Язык реализации: Python 3.11+.
- Использование только стандартной библиотеки Python (`urllib`, `json`, `argparse`, `datetime`, `sys`, `pathlib`, `tempfile`, `socket`).
- Кодировка ввода-вывода: UTF-8.

### 1.4. Границы решения

- Только курс USD/RUB.
- Только консольный вывод, без UI, баз данных и истории.
- Без аутентификации, API-ключей и персональных данных.

## 2. Функциональные требования (FR-NN)

### FR-01 — Получение курса от приоритетного источника

Скрипт должен выполнять HTTP GET-запрос к основному источнику:

```text
https://www.cbr-xml-daily.ru/daily_json.js
```

и извлекать значение курса USD/RUB из поля `Valute.USD.Value`.

**Важно:** Данный endpoint является **неофициальным mirror'ом** курсов ЦБ РФ. Для справочного использования допустимо; для юридически значимых или финансово критичных расчётов рекомендуется использовать официальный API ЦБ РФ.

**Критерии приёмки:**
- При успешном ответе используется курс из неофициального mirror'а.
- Дата актуальности берётся из поля `Date` ответа.
- Приоритет источника: mirror ЦБ РФ > fallback.

### FR-02 — Использование fallback-источника

Если основной источник недоступен или вернул некорректные данные, скрипт должен выполнить запрос к fallback-источнику:

```text
https://open.er-api.com/v6/latest/USD
```

и извлекать курс из поля `rates.RUB`.

**Критерии приёмки:**
- Fallback используется только при ошибке основного источника.
- Если fallback недоступен, скрипт завершается с ошибкой.
- Дата актуальности берётся из поля `time_last_update_utc`.

### FR-03 — Консольный вывод результата

Скрипт должен выводить результат в формате:

```text
USD/RUB: <курс> (источник: <источник>, дата: <дата_время>)
```

**Критерии приёмки:**
- Курс форматируется с двумя знаками после запятой.
- Источник указывается понятным именем (`ЦБ РФ` или `open.er-api.com`).
- Дата приводится к московскому времени (MSK, UTC+3).

### FR-04 — Разбор аргументов командной строки

Скрипт должен поддерживать аргументы командной строки, перечисленные в разделе [CLI-аргументы](#5-cli-аргументы).

**Критерии приёмки:**
- Некорректные аргументы вызывают вывод справки и код возврата `2`.
- `--help` выводит справку и завершает скрипт с кодом `0`.

### FR-05 — Кэширование последнего успешного результата

Скрипт должен поддерживать файловый кэш последнего успешного ответа.

**Критерии приёмки:**
- По умолчанию кэш читается и записывается.
- TTL кэша по умолчанию: 5 минут.
- Флаг `--no-cache` отключает чтение и запись кэша.
- Флаг `--refresh` игнорирует кэш и обновляет данные.

### FR-06 — Использование просроченного кэша

При недоступности сети скрипт может вывести просроченный кэш, если явно указан флаг `--use-stale`.

**Критерии приёмки:**
- Без флага `--use-stale` просроченный кэш не используется.
- При использовании stale-кэша в вывод добавляется пометка `(кэш)`.

### FR-07 — Подробный вывод

При указании флага `--verbose` скрипт должен выводить диагностику по каждому источнику.

**Критерии приёмки:**
- Для каждого источника выводится статус (`OK` / `FAILED`) и причина отказа.
- Диагностика выводится до основного результата.

### FR-08 — Обработка ошибок

При невозможности получить данные из всех источников скрипт должен вывести понятное сообщение об ошибке в `stderr` и завершиться с кодом `1`.

**Критерии приёмки:**
- Сообщение по умолчанию:
  ```text
  Не удалось получить курс USD/RUB. Проверьте подключение к интернету.
  ```
- По умолчанию traceback не выводится.
- В режиме `--verbose` может выводиться краткая диагностика.

## 3. Системные требования (SR-NN)

### SR-01 — Версия Python

Для запуска скрипта требуется интерпретатор Python 3.11 или новее.

### SR-02 — Стандартная библиотека

Скрипт должен работать исключительно на стандартной библиотеке Python. Установка пакетов через `pip` не требуется.

### SR-03 — Сетевой доступ

Устройство должно иметь доступ в интернет для выполнения HTTP-запросов к источникам данных.

### SR-04 — Таймауты

Таймаут HTTP-запроса по умолчанию составляет 10 секунд и может быть переопределён через аргумент `--timeout`.

### SR-05 — Кодировка

Все входные и выходные потоки должны использовать кодировку UTF-8.

### SR-06 — Размер файла

Размер исходного файла `usd_rub_rate.py` не должен превышать 50 КБ.

### SR-07 — Коды возврата

Скрипт должен завершаться с одним из следующих кодов:

| Код | Значение |
|-----|----------|
| `0` | Успех, курс получен и выведен |
| `1` | Ошибка получения данных из всех источников |
| `2` | Ошибка разбора аргументов командной строки |

### SR-08 — Расположение кэша

Кэш хранится в JSON-файле по пути:

```text
~/.cache/usd-rub-rate/cache.json
```

либо, при отсутствии домашней директории, рядом со скриптом:

```text
./.usd-rub-cache.json
```

Директория кэша создаётся автоматически при необходимости.

### SR-09 — Поддержка часового пояса

Временные метки fallback-источника из UTC должны преобразовываться в московское время (MSK, UTC+3).

### SR-10 — Логирование

В режиме `--verbose` скрипт выводит диагностику в `stderr` или `stdout`. В обычном режиме посторонний вывод не допускается.

## 4. Контракты API источников

### 4.1. Основной источник: cbr-xml-daily.ru (неофициальный mirror)

**URL:**

```text
GET https://www.cbr-xml-daily.ru/daily_json.js
```

**Примечание:** Этот endpoint предоставляется **неофициальным mirror'ом** курсов ЦБ РФ (сторонний сервис, Cloudflare). Используется для удобства JSON-парсинга; для официальных целей — `https://www.cbr.ru/scripts/XML_daily.asp`.

**Заголовки:**

| Заголовок | Значение |
|-----------|----------|
| `Accept` | `application/json` |
| `User-Agent` | `usd-rub-rate/<version>` |

**Пример ответа (HTTP 200):**

```json
{
  "Date": "2026-07-19T11:30:00+03:00",
  "PreviousDate": "2026-07-18T11:30:00+03:00",
  "PreviousURL": "//www.cbr-xml-daily.ru/archive/...",
  "Timestamp": "2026-07-18T20:00:00+03:00",
  "Valute": {
    "USD": {
      "ID": "R01235",
      "NumCode": "840",
      "CharCode": "USD",
      "Nominal": 1,
      "Name": "Доллар США",
      "Value": 92.4567,
      "Previous": 92.1234
    }
  }
}
```

**Используемые поля:**

| Поле | Тип | Назначение |
|------|-----|------------|
| `Valute.USD.Value` | `float` | Курс USD/RUB |
| `Date` | `string` (ISO-8601) | Дата актуальности по московскому времени |

**Стратегия fallback:**
- HTTP >= 400
- Таймаут
- Некорректный JSON
- Отсутствует `Valute.USD.Value`

## 5. CLI-аргументы

| Аргумент | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `--source` | `str` | `auto` | Принудительный выбор источника: `cbr`, `fallback`, `auto` |
| `--timeout` | `float` | `10.0` | Таймаут HTTP-запроса в секундах |
| `--no-fallback` | `flag` | `False` | Не использовать fallback-источник |
| `--no-cache` | `flag` | `False` | Отключить чтение и запись кэша |
| `--refresh` | `flag` | `False` | Игнорировать кэш и обновить данные |
| `--use-stale` | `flag` | `False` | При сетевой ошибке использовать просроченный кэш |
| `--verbose` | `flag` | `False` | Подробный вывод диагностики |
| `--version` | `flag` | `False` | Показать версию скрипта и завершиться с кодом `0` |
| `-h`, `--help` | `flag` | `False` | Показать справку и завершиться с кодом `0` |

**Примеры запуска:**

```bash
python usd_rub_rate.py
python usd_rub_rate.py --source cbr --timeout 5
python usd_rub_rate.py --refresh --verbose
python usd_rub_rate.py --no-fallback
python usd_rub_rate.py --use-stale
```

**Ограничения:**
- `--no-cache` и `--refresh` могут использоваться одновременно.
- `--use-stale` игнорируется при наличии валидного кэша или успешном сетевом запросе.
- `--source fallback` отключает попытку запроса к ЦБ РФ.
- `--source cbr` в сочетании с `--no-fallback` приводит к ошибке, если ЦБ РФ недоступен.

## 6. Схема JSON-файла кэша

### 6.1. Расположение

```text
~/.cache/usd-rub-rate/cache.json
```

### 6.2. Структура

```json
{
  "rate": 92.46,
  "source": "cbr",
  "source_name": "ЦБ РФ",
  "timestamp": "2026-07-19T11:30:00+03:00",
  "cached_at": "2026-07-19T11:32:15+03:00"
}
```

### 6.3. Поля

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `rate` | `float` | Да | Курс USD/RUB |
| `source` | `string` | Да | Идентификатор источника (`cbr` или `fallback`) |
| `source_name` | `string` | Да | Человекочитаемое имя источника |
| `timestamp` | `string` | Да | Дата актуальности курса (ISO-8601 с таймзоной) |
| `cached_at` | `string` | Да | Дата записи кэша (ISO-8601 с таймзоной) |

### 6.4. TTL

- По умолчанию: 300 секунд (5 минут).
- При TTL > 0 кэш считается валидным.
- При `cached_at` в будущем или отсутствии полей кэш считается невалидным.

### 6.5. Поведение

- При валидном кэше и отсутствии `--refresh` данные не запрашиваются по сети.
- После успешного сетевого запроса кэш перезаписывается (если не указан `--no-cache`).
- При использовании stale-кэша значение `cached_at` не обновляется.

**Успех основного источника:**

```text

USD/RUB: 92.46 (источник: ЦБ РФ, дата: 2026-07-19 11:30:00 MSK)
```

**Fallback:**

```text
[ЦБ РФ] FAILED: HTTP 503
[open.er-api.com] OK: 92.4500
USD/RUB: 92.45 (источник: open.er-api.com, дата: 2026-07-19 03:00:00 MSK)
```

## 8. Матрица обработки ошибок

| # | Сценарий | Поведение | Код возврата |
|---|----------|-----------|--------------|
| E1 | Успешный ответ от ЦБ РФ | Вывод курса | `0` |
| E2 | Таймаут основного источника | Переход к fallback | `0` при успехе |
| E3 | HTTP 4xx/5xx от основного источника | Переход к fallback | `0` при успехе |
| E4 | Некорректный JSON от основного источника | Переход к fallback | `0` при успехе |
| E5 | Отсутствует `Valute.USD.Value` | Переход к fallback | `0` при успехе |
| E6 | Недоступен fallback | Вывод ошибки в `stderr` | `1` |
| E7 | Некорректный JSON от fallback | Вывод ошибки в `stderr` | `1` |
| E8 | Отсутствует `rates.RUB` | Вывод ошибки в `stderr` | `1` |
| E9 | Отсутствие интернета | Вывод ошибки в `stderr` | `1` |
| E10 | Ошибка разбора аргументов | Вывод справки в `stderr` | `2` |
| E11 | Валидный кэш, сеть недоступна | Вывод кэша | `0` |
| E12 | Просроченный кэш + `--use-stale` | Вывод кэша с пометкой | `0` |
| E13 | Просроченный кэш без `--use-stale` | Вывод ошибки | `1` |

**Тексты сообщений об ошибках:**

| Код | Сообщение |
|-----|-----------|
| E6–E9, E13 | `Не удалось получить курс USD/RUB. Проверьте подключение к интернету.` |
| E10 | Справка `argparse` |

**Дополнительно в `--verbose`:**
- Для каждого источника выводится строка вида `[<имя>] FAILED: <причина>`.
- Причины: `HTTP <код>`, `timeout`, `invalid JSON`, `missing field`, `network error`.

## 9. Критерии приёмки

### AC-01 — Базовый запуск

При запуске `python usd_rub_rate.py` в терминале выводится строка с курсом USD/RUB, источником и датой.

### AC-02 — Приоритет ЦБ РФ

При доступности обоих источников используется курс ЦБ РФ.

### AC-03 — Fallback

При недоступности ЦБ РФ автоматически используется fallback-источник.

### AC-04 — Отключение fallback

При запуске с `--no-fallback` и недоступности ЦБ РФ скрипт завершается с ошибкой (код `1`).

### AC-05 — Форматирование

Курс всегда выводится с двумя знаками после запятой.

### AC-06 — Часовой пояс

Дата в выводе указана по московскому времени (MSK).

### AC-07 — Кэширование

При повторном запуске в течение 5 минут скрипт использует кэш (если не указан `--refresh`).

### AC-08 — Просроченный кэш

При сетевой недоступности и флаге `--use-stale` выводится последний известный курс с пометкой `(кэш)`.

### AC-09 — Понятные ошибки

При любой неуспешной попытке получения данных выводится понятное сообщение, а не traceback.

### AC-10 — Stdlib-only

Скрипт не импортирует сторонние пакеты и не требует `pip install`.

## 10. Нефункциональные требования

### NFR-01 — Производительность

Время от запуска до вывода результата не более 10 секунд при нормальном интернет-соединении.

### NFR-02 — Надёжность

Скрипт корректно завершается при недоступности API без traceback по умолчанию.

### NFR-03 — Безопасность

Скрипт не хранит и не передаёт персональные данные, не использует API-ключи.

### NFR-04 — Потребление ресурсов

Размер исходного файла не превышает 50 КБ.
