# AccountsHelper — Business Requirements Document

## 1. Общие сведения

| Параметр | Значение |
|---|---|
| Название продукта | AccountsHelper |
| Тип | Расширение для браузера Google Chrome |
| Цель | Ускорить заполнение повторяющихся web-форм (airdrop, whitelist, waitlist, регистрации) за счёт хранения личных данных пользователя в зашифрованном локальном хранилище и вставки их через контекстное меню по правой кнопке мыши. |
| Целевая аудитория | Пользователи, часто заполняющие формы с email, кошельками, социальными сетями и контактами. |
| Язык интерфейса | Русский / English (MVP — русский, i18n заложить архитектурно). |

## 2. MVP Scope

### 2.1 Возможности расширения

1. **Профиль пользователя** — сохранение в зашифрованном виде:
   - Email (один или несколько)
   - EVM-адреса кошельков (Ethereum, BSC, Polygon, Arbitrum, Optimism и др. — сети семейства EVM)
   - Bitcoin (BTC) адреса
   - Discord handle (`@username`)
   - Telegram handle (`@username`)
   - X / Twitter handle (`@username`)
   - Телефон
   - Имя (First Name / Last Name)
   - Nickname

2. **PIN-защита** — расширение разблокируется мастер-PIN перед показом данных или вставкой.

3. **Контекстное меню (ПКМ)** — при клике правой кнопкой на поле ввода на странице:
   - Автоопределение типа поля по атрибутам: `name`, `id`, `placeholder`, `autocomplete`, `aria-label`, `inputmode`, `type`, соседние `<label>`.
   - Предложение подходящих значений из сохранённого профиля.
   - Возможность вставить любое другое сохранённое значение через подменю.

4. **Popup расширения** — управление профилем:
   - Добавление / редактирование / удаление записей.
   - Блокировка / разблокировка PIN.
   - Импорт / экспорт профиля (MVP — экспорт в JSON, импорт из JSON; оба зашифрованы при необходимости).

5. **Автоподстановка (опционально для MVP, если успеваем)** — при загрузке страницы автоматически заполнить все распознанные поля, если расширение разблокировано.

### 2.2 Out of MVP

- Синхронизация между устройствами (chrome.storage.sync) без собственного шифрования.
- Бэкенд / облачное хранилище.
- Поддержка Firefox / Safari.
- Web3-подпись транзакций (только хранение адресов).
- OCR / распознавание полей по визуальному расположению (без атрибутов).

## 3. Функциональные требования

### FR-1. Управление PIN
- Пользователь устанавливает PIN при первом запуске.
- PIN используется для шифрования ключа шифрования профиля (временный session key).
- После N неправильных попыток (по умолчанию 5) профиль временно блокируется на время (экспоненциальная задержка).
- Возможность сброса PIN с полным стиранием данных.

### FR-2. Управление данными профиля
- CRUD для каждого типа данных.
- Валидация форматов: email, EVM-address, BTC address, phone (E.164 или произвольно), handles (`@username`).
- Возможность задать псевдоним для каждой записи (например, «Основной MetaMask», «Кошелёк для дропов»).
- Пометка записи по умолчанию для быстрой вставки.

### FR-3. Автоопределение полей
- При ПКМ на `<input>`, `<textarea>` или `[contenteditable]` расширение анализирует поле и окружение.
- Распознаваемые типы полей:
  - `email` → Email
  - `evm_address` / `wallet` → EVM-адреса
  - `btc_address` → Bitcoin-адреса
  - `discord` → Discord handle
  - `telegram` → Telegram handle
  - `twitter` / `x` → X handle
  - `phone` / `tel` → Телефон
  - `first_name`, `last_name`, `full_name` → Имя
  - `nickname` / `username` → Nickname
- Если тип не определён, в контекстном меню показывается полный список категорий.

### FR-4. Контекстное меню
- Главное меню: «AccountsHelper →».
- Подменю при совпадении типа: «Вставить email», «Вставить @telegram» и т.д.
- Если совпадений несколько (например, несколько email), показывается список с псевдонимами.
- При заблокированном расширении пункт меню ведёт на разблокировку (открытие popup).

### FR-5. Вставка значения
- Вставка через `element.value = value` или `element.textContent` для contenteditable.
- Триггер событий `input` и `change` для совместимости с React / Vue / Angular формами.
- Визуальная обратная связь: подсветка поля на 1 секунду после вставки.

### FR-6. Экспорт / импорт
- Экспорт зашифрованного JSON с профилем.
- Импорт зашифрованного JSON (с запросом PIN).

## 4. Нефункциональные требования

### NFR-1. Безопасность
- Все чувствительные данные хранятся в `chrome.storage.local` в зашифрованном виде (AES-256-GCM).
- Ключ шифрования держится в memory service worker только после разблокировки и удаляется при блокировке / таймауте.
- PIN не хранится в открытом виде; проверка через Argon2id / PBKDF2 (обоснование в HLD).
- Запрет eval / inline scripts; CSP строгий.
- Никаких внешних сетевых запросов от имени расширения (кроме обновления, если потребуется).

### NFR-2. Производительность
- Время появления контекстного меню < 200 мс.
- Расшифровка профиля < 500 мс на современном устройстве.

### NFR-3. Надёжность
- При ошибке шифрования / дешифрования показывать понятное сообщение, не падать.
- Fallback: если автоопределение не сработало, дать пользователю выбрать вручную.

### NFR-4. UX
- Минимум кликов: ПКМ → категория → вставить.
- Интуитивный popup с табами / секциями.
- Подсказки при первом использовании.

## 7. Пользовательские сценарии

### US-1. Первый запуск
1. Пользователь устанавливает расширение.
2. Открывается popup с просьбой установить PIN.
3. Пользователь вводит PIN дважды.
4. Пользователь добавляет email и один EVM-адрес.

### US-2. Заполнение формы airdrop
1. Пользователь открывает форму на сайте.
2. Кликает ПКМ на поле «Enter your wallet address».
3. Видит «AccountsHelper → Вставить EVM-адрес → Основной MetaMask».
4. Кликает — адрес вставляется.

### US-3. Быстрая вставка в незнакомое поле
1. Пользователь ПКМ на поле без понятных атрибутов.
2. Выбирает «AccountsHelper → Все данные → Email → Рабочий email».
3. Значение вставляется.

### US-4. Экспорт перед переустановкой
1. Пользователь открывает popup → Настройки → Экспорт.
2. Вводит PIN.
3. Скачивает зашифрованный JSON.

## 8. Критерии приёмки MVP

- [ ] Установка расширения из архива `.zip` / Chrome Web Store (после публикации).
- [ ] Установка PIN, разблокировка, смена PIN, сброс с удалением данных.
- [ ] CRUD всех типов данных из п. 2.1.
- [ ] Валидация форматов email, EVM, BTC, phone, handles.
- [ ] ПКМ-меню появляется на полях ввода.
- [ ] Автоопределение работает на тестовых страницах (Google Forms, Typeform, произвольная HTML-форма).
- [ ] Вставка корректно триггерит события input/change для React/Vue/Angular.
- [ ] Данные остаются зашифрованными в `chrome.storage.local`.
- [ ] Экспорт/импорт профиля.

## 9. Риски

- Chrome Web Store может отклонить расширение из-за шифрования / хранения адресов кошельков. Нужно чёткое описание в privacy policy.
- Web Crypto API в service worker MV3 имеет ограничения (нет `window.crypto.subtle` в некоторых контекстах; нужно проверить и адаптировать).
- Content script injection должен быть максимально безопасным, чтобы не считался malware.
